##  Innovative UX 

![[07-01-02_04_UX Prompt.pdf]]

### Implementierung einer innovativen UX – Beispiel-Prompt

Probieren Sie diesen Prompt mit einer etwas ungewöhnlichen UX-Idee in einem neuen Projekt aus:

Eine Web-App, die aus zwei Spalten besteht:

- **Linke Spalte:** Hier wird das Transkript eines Podcasts angezeigt. Innerhalb des Transkripts gibt es markierte Abschnitte mit Bereichsmarkierungen am Anfang und Ende. Diese Markierungen können per Drag-and-Drop verschoben werden, um die Größe des markierten Abschnitts zu verändern. Markierte Abschnitte können sich gegenseitig überlappen.
    
- **Rechte Spalte:** Jeder markierte Abschnitt ist mit einem Beitragsvorschlag in der rechten Spalte verbunden. Der Vorschlag zeigt an, für welches Medium der Beitrag gedacht ist (LinkedIn, Twitter, Blog) und enthält einige Stichpunkte zur Beitragsidee.
    
- **Darstellung:** Jeder Beitragsvorschlag wird vertikal neben dem zugehörigen markierten Abschnitt ausgerichtet, wobei eine gestrichelte Linie beide verbindet. Beitragsvorschläge in der rechten Spalte dürfen sich nicht überlappen; sie werden einfach übereinander gestapelt. Wenn sie kollidieren, verschieben sie sich nach oben oder unten.

### Praktische Übung

### Brainstorming eigener Ideen

Es muss nicht gleich das nächste Instagram sein, das die Welt komplett verändert. Es kann etwas Kleines und Nischiges sein – Hauptsache ist, dass Sie anfangen.

1. Wir wollen immer ein spezifisches Nutzerproblem lösen, sonst wird niemand Ihre App benutzen.    
2. Wie findet man Nutzerprobleme? Fangen Sie bei sich selbst an. Was sind Ihre Hobbys? Was nervt Sie im Arbeitsalltag? Wie könnten Sie Ihr Privatleben angenehmer gestalten?
    
3. Können Sie die 3 Ws beantworten?    
    - Was ist das Problem?        
    - Wer hat das Problem?        
    - Wie groß ist das Problem?
        
4. Sobald Sie einen Schmerzpunkt (Pain Point) identifiziert haben, überlegen Sie, ob dieser durch eine App gelöst werden kann.    
5. Wenn ja: Weitermachen. Wenn nein: Zurück zu Punkt 2.

##  Landing Page Prompt

![[07-01-02_06_Landing Page.pdf]]

Entwickeln Sie eine Landing Page für ein Software-as-a-Service (SaaS) Produkt, das Gärtnern dabei hilft zu entscheiden, welche Gemüsesorten auf Feldern gepflanzt werden sollen, die im Laufe der Saison frei werden.

Die Landing Page sollte die folgenden Elemente enthalten:

- **Die 3 Ws:** Kommunizieren Sie klar, was das Problem ist, wer das Problem hat und wie groß das Problem ist.    
- **Preisübersicht:** Es sollte eine Übersicht über die Preisgestaltung geben.    
- **Team-Vorstellung:** Das Team hinter dem Produkt sollte vorgestellt werden.
    
Kontext zum Problem

- **Wer hat das Problem?** Ich bin ein Gärtner.    
- **Ausgangslage:** Zu Beginn der Saison pflanze ich das an, was ich im Winter geplant habe. Im Laufe der Saison werden dann Felder frei.    
- **Was ist das Problem?** Ich würde gerne neues Gemüse anpflanzen, weiß aber nicht, was ich wann säen kann.    
- **Wie groß ist das Problem?** Das führt dazu, dass Felder ungenutzt und leer bleiben, austrocknen und sich die Bodenqualität verschlechtert. Vor allem aber fällt meine Ernte etwa 30 % geringer aus, als sie sein könnte.
